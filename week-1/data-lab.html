01

Create & Clone

Quick review of week one — a fresh repo for a fresh project.

1

Create your repository on GitHub

Goal

Have a remote home for this new project before touching a terminal.

Requirements

  • A new, empty repository exists — name it something like tirana-practice (you'll reuse this same repo through Week 3)
  • No README, .gitignore, or license was added at creation
  • You have copied its HTTPS clone URL

Do this

  1. Go to github.com/new
  2. Name it tirana-practice
  3. Leave every checkbox unchecked — keep it empty
  4. Click Create repository, then copy the HTTPS URL from the green Code button
2

Clone it with your token

Goal

Get a connected local folder, authenticated from the first command.

Requirements

  • You have a classic Personal Access Token (repo scope) ready
  • A local folder exists, connected to your repo as origin

Swap in your own username, token, and repo name

git clone https://<username>:<token>@github.com/<username>/tirana-practice.git
cd tirana-practice
3

Add a README and make your first commit

Goal

Start with a one-line description and a clean first snapshot.

Requirements

  • README.md has a title and one sentence describing the project
  • git log shows one commit, pushed to GitHub
  1. touch README.md, then open it and add a title plus one sentence
git add .
git commit -m "initial commit"
git push origin main
02

Folder Structure

Separate raw data, code, and generated results — a convention you'll see in every real project from here on.

4

Create your project folders

Goal

Set up three folders that keep raw data, scripts, and generated output cleanly separated.

Requirements

  • data/, scripts/, and output/ all exist at the project root
mkdir data scripts output
ls -la
5

Make Git track your empty folders

Goal

Git never tracks empty folders — a small placeholder file is the standard trick to make a folder structure show up on GitHub immediately.

Requirements

  • Each of data/, scripts/, and output/ contains a .gitkeep file
touch data/.gitkeep scripts/.gitkeep output/.gitkeep
6

Commit your project structure

Goal

Save the folder layout, and start a .gitignore you'll build on in the next phase.

Requirements

  • .gitignore exists at the project root
  • GitHub shows the data/, scripts/, and output/ folders
touch .gitignore
git add .
git commit -m "add project folder structure"
git push
03

Virtual Environment

Give this project its own isolated Python — separate from your system and every other project.

7

Create a virtual environment

Goal

Create an isolated Python environment that lives entirely inside this project folder.

Requirements

  • A venv/ folder now exists at the project root
python3 -m venv venv
8

Activate it and confirm it's active

Goal

Make sure every install from here on lands inside this project's environment, not your system Python.

Requirements

  • Your terminal prompt now shows (venv) at the start of the line
  • which python points to a path inside your venv folder
source venv/bin/activate
which python
9

Ignore your virtual environment in Git

Goal

A venv folder is large and machine-specific — it should never be committed.

Requirements

  • .gitignore contains a line with venv/
  • git status no longer lists anything inside venv/ as untracked
echo "venv/" >> .gitignore
git add .gitignore
git commit -m "ignore virtual environment"
git push
04

Install Libraries

Add the two libraries this whole session depends on.

10

Install pandas and matplotlib

Goal

Install the two data libraries — inside your virtual environment, not system-wide.

Requirements

  • (venv) is still showing in your prompt
  • pip show pandas and pip show matplotlib both return version info
pip install pandas matplotlib
11

Freeze your dependencies

Goal

Record exactly what's installed, so anyone — including future you — can recreate this environment.

Requirements

  • requirements.txt exists and lists pandas and matplotlib with version numbers
pip freeze > requirements.txt
cat requirements.txt
12

Commit requirements.txt

Goal

Save the dependency list — never the venv folder itself.

Requirements

  • GitHub shows requirements.txt, but no venv folder
git add requirements.txt
git commit -m "add project dependencies"
git push

Generate Fake Data

A real data file in data/, written by AI, run by you.

13

Ask AI to generate the fake dataset

Goal

Get a realistic-looking CSV of fake apartments saved into data/.

Requirements

  • scripts/generate_data.py exists
  • Running it creates data/apartments.csv with at least 30 rows
  • Columns include at least price, sqm, and bedrooms

Say this to your AI assistant

"Write a Python script that generates 40 fake Tirana apartment listings with columns: price (in EUR), sqm, bedrooms, bathrooms, floor, and neighborhood. Save it as data/apartments.csv using pandas. Print a confirmation message with the number of rows saved."

14

Goal

Save both the code that generated the data and the data itself.

Requirements

git push
