01

Create & Clone

Give the project a home on GitHub before a single local file exists.

1

Create your repository on GitHub

Goal

Have a remote home for the project waiting before you touch a terminal.

Requirements

  • A new, empty repository named tirana-scout exists on your GitHub account
  • No README, .gitignore, or license was added at creation
  • You have copied its HTTPS clone URL

Do this

  1. Go to github.com/new in your browser
  2. Name the repository tirana-scout
  3. Leave every checkbox unchecked — keep it empty
  4. Click Create repository, then copy the HTTPS URL from the green Code button
2

Clone it to your machine

Goal

Turn that empty remote repository into a connected local folder you can actually work in.

Requirements

  • You have a classic Personal Access Token (repo scope) copied and ready
  • A local folder named tirana-scout exists on your machine
  • Running git remote -v inside it shows an origin pointing at your GitHub repo

Swap in your own username, token, and repo name

git clone https://<username>:<token>@github.com/<username>/tirana-scout.git
cd tirana-scout
02

Terminal Basics

Create files and check your surroundings — no file explorer allowed.

3

Create your first files

Goal

Practice creating files from the terminal instead of right-clicking in a file browser.

Requirements

  • app.py and README.md exist inside tirana-scout
  • Listing the folder's contents shows both files
touch app.py README.md
ls -la
4

Confirm where you are

Goal

Get comfortable checking your location and peeking inside a file before running commands blindly.

Requirements

  • You can print the full path of your current folder
  • You can view the (empty) contents of README.md without opening an editor
pwd
cat README.md
03

First Commit & Push

Turn two empty files into your first real snapshot of history.

5

See what Git notices

Goal

See how Git tracks new files before anything is committed.

Requirements

  • Git reports both files as untracked
git status
6

Stage and commit

Goal

Turn untracked files into your first permanent snapshot of project history.

Requirements

  • git status reports a clean working tree
  • git log shows exactly one commit
git add .
git commit -m "initial project structure"
7

Push to GitHub

Goal

Get your first commit off your machine and onto the remote you created in Phase 1.

Requirements

  • Refreshing the repository page on GitHub shows app.py and README.md
git push origin main
04

Feature: Listings

Your first feature — written by describing it, not by typing it.

8

Ask your AI assistant for the first feature

Goal

Get working Python written for you by describing the outcome you want.

Requirements

  • app.py holds a list of at least 5 apartment records (price, sqm, bedrooms)
  • A function prints them in a readable format
  • Running the file prints all 5 listings

Say this to your AI assistant

"Write a Python function that stores 5 fake Tirana apartments as a list of dictionaries with price, sqm, and bedrooms, and a function that prints them in a readable format. Add a small block at the bottom that calls it."

python3 app.py
9

Save the checkpoint

Goal

Turn this working feature into a point you can always return to.

Requirements

  • git log shows a new commit for this feature
  • GitHub shows the updated app.py
git add app.py
git commit -m "add listings and print function"
git push
05

Feature: Filter

Build on working code instead of starting over.

10

Ask for a price filter

Goal

Add a second feature by prompting again, on top of code that already works.

Requirements

  • app.py has a function that filters listings under a given max price
  • Calling it with two different max prices gives two different results

Say this to your AI assistant

"Add a function to my existing code that filters the apartment list to only show listings under a given max price. Call it with two different values and print both results."

python3 app.py
11

Commit and push again

Goal

Build the habit — every working feature earns its own commit.

Requirements

  • git log now shows a commit mentioning the filter
git add app.py
git commit -m "add price filter"
git push
06

Feature: Deal Score

A small preview of the idea behind the real three-week project.

12

Ask for a deal-scoring function

Goal

Add the idea that mirrors what you'll build for real over the next three weeks: labeling listings as good or bad deals.

Requirements

  • app.py computes price-per-square-meter for each listing
  • Each listing is labeled good deal or bad deal against the average

Say this to your AI assistant

"Add a function that calculates price per square meter for each apartment, compares it to the average across all listings, and labels each one 'good deal' or 'bad deal'. Print the results."

python3 app.py
13

Lock in the last feature

Goal

Commit the final feature of the build.

Requirements

  • git log shows a commit mentioning deal scoring
git add app.py
git commit -m "add deal scoring"
git push
07

Wrap-up

Everything you just did is recorded, in order, forever.

14

See your build as a timeline

Goal

Look back at the whole hour as a single, ordered history.

Requirements

  • Every commit from this session appears, in order, on one screen
git log --oneline
15

Tag this milestone

Goal

Mark this working version so anyone can return to exactly this state, later.

Requirements

  • git tag lists v1.0
  • GitHub's repository page shows one tag
git tag v1.0
git push --tags
16

Look at what you built

Goal

Open your repository on GitHub and scroll through the commit history together. Sixteen small steps, a working script, and a real Git timeline — built by prompting, not typing.
