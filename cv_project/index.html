01

Create & Clone

Give the project a home on GitHub before a single local file exists.

1

Create your repository on GitHub

Goal

Have a remote home for your CV waiting before you touch a terminal.

Requirements

  • A new, empty repository exists on your GitHub account — name it something like my-cv
  • No README, .gitignore, or license was added at creation
  • You have copied its HTTPS clone URL

Do this

  1. Go to github.com/new in your browser
  2. Name it my-cv (or anything personal — it's your repo)
  3. Leave every checkbox unchecked — keep it empty
  4. Click Create repository, then copy the HTTPS URL from the green Code button
2

Clone it with your token

Goal

Turn the empty remote repository into a connected local folder — authenticated from the very first command.

Requirements

  • You have a classic Personal Access Token (repo scope) copied and ready
  • A local folder exists on your machine, connected to your GitHub repo as origin

Swap in your own username, token, and repo name

git clone https://<username>:<token>@github.com/<username>/my-cv.git
cd my-cv
3

Create your starter files

Goal

Set up the three files this whole project lives in, plus a README to describe it.

Requirements

  • index.html, styles.css, script.js, and README.md all exist in the folder
  • Listing the folder's contents shows all four files
touch index.html styles.css script.js README.md
ls -la
4

Write a one-line README

Goal

Get in the habit of describing a project before you build it — even in one sentence.

Requirements

  • README.md contains at least a title and one sentence describing the project
  • cat README.md prints what you wrote
  1. Open README.md in VS Code
  2. Type something like: # My CV on the first line, then A personal CV site, built one commit at a time. on the next
  3. Save, then confirm with cat README.md in the terminal
5

First commit and push

Goal

Get your very first snapshot onto GitHub before any real content exists.

Requirements

  • git log shows exactly one commit
  • Refreshing your repository page on GitHub shows all four files
git add .
git commit -m "initial project structure"
git push origin main

07

Deploy & Wrap-up

Ship it — using the exact same skill you'll use for every project after this one.

17

Enable GitHub Pages

Goal

Turn your repository into a live, public website with no extra hosting setup.

Requirements

  • Your repository's Pages settings show a green "Your site is live at..." banner
  • Opening that link shows your actual CV, styled and working
  1. On GitHub, open your repository's Settings tab
  2. Go to Pages in the left sidebar
  3. Under "Build and deployment," choose Deploy from a branch
  4. Set branch to main and folder to / (root), then Save
  5. Wait about a minute, then refresh the page for your live link
18

Tag the milestone and view your history

Goal

Mark this working version, then look back at the whole build as one ordered history.

Requirements

  • git tag lists v1.0
  • git log --oneline shows every commit from this session, in order
git tag v1.0
git push --tags
git log --oneline
19
